## Getting Started

If you are a new member, follow these steps in order:

1. **Review the [Wider Team Integration](Wider-Team-Integration.md)** guide to understand the 3-phase approach for joining the team.
2. **Read the [Onboarding Materials](Onboarding%20Materials.md)** - covers all tools, workflows, and expectations.
3. **Set up your tools** by following the onboarding guide:
   - **Slack** - team communication and daily status updates
   - **GitHub** - code repositories and project management
   - **FigJam** - whiteboarding, brainstorming, and sprint planning
   - **Obsidian** - this vault, for markdown-based documentation and wiki-linking
4. **Explore your assigned project materials** :
   - [Mujarrad](https://github.com/Wider-Community/Wider-Team-Board/tree/main/Mujarrad)
   - [QUD](https://github.com/Wider-Community/Wider-Team-Board/tree/main/QUD%C2%B3)
   - [AQWUM](https://github.com/Wider-Community/Wider-Team-Board/tree/main/QUD%C2%B3)

## Template SDLC

The **[Template SDLC](Template%20SDLC.canvas)** canvas provides a reusable template for writing the Software Development Life Cycle documentation for any project. It serves as the standard format for defining project phases, deliverables, and milestones. Open it in Obsidian to view and adapt it for your project.

A project-specific example is available in the Slack Bot Agent: [Slack Agent SDLC](Agentic%20Systems/Slack%20Bot%20Agent/Slack%20Agent%20SDLC.canvas).
