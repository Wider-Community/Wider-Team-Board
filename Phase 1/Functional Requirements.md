## 1 Async Standup Management

**FR-1.1.1** The system SHALL run scheduled daily async standups in configured channels  
**FR-1.1.2** The system SHALL prompt team members with customizable questions (what worked on, what's next, blockers)  
**FR-1.1.3** The system SHALL collect responses via DM or thread replies  
**FR-1.1.4** The system SHALL generate and post daily standup summaries  
**FR-1.1.5** The system SHALL support follow-up questions for incomplete responses  
**FR-1.1.6** The system SHALL allow per-channel standup configuration (timing, questions, participants)

## 2 Task Tracking & Follow-ups

**FR-2.1** The system SHALL identify action items from conversations  
**FR-2.2** The system SHALL extract task assignments and deadlines from messages  
**FR-2.3** The system SHALL send automated reminders for pending tasks  
**FR-2.4** The system SHALL track task status (Open, In Progress, Blocked, Completed)  
**FR-2.5** The system SHALL escalate overdue tasks after configurable timeouts  
**FR-2.6** The system SHALL support manual task creation and updates

## 3 Channel & Thread Monitoring

**FR-3.1** The system SHALL monitor configured channels for mentions  
**FR-3.2** The system SHALL track thread participation and activity  
**FR-3.3** The system SHALL maintain conversation context per thread  
**FR-3.4** The system SHALL respond contextually within threads  
**FR-3.5** The system SHALL identify unanswered questions in threads  
**FR-3.6** The system SHALL detect when discussions become stale

## 4 Discussion Summaries

**FR-4.1** The system SHALL summarize long threads on-demand or automatically  
**FR-4.2** The system SHALL extract key points from discussions  
**FR-4.3** The system SHALL identify decisions made in conversations  
**FR-4.4** The system SHALL capture action items from discussions  
**FR-4.5** The system SHALL list open questions  
**FR-4.6** The system SHALL post summaries to threads or designated channels

## 5 Reminders & Nudges

**FR-5.1** The system SHALL send proactive nudges for unanswered questions  
**FR-5.2** The system SHALL remind users about missed standups  
**FR-5.3** The system SHALL follow up on pending confirmations  
**FR-5.4** The system SHALL support configurable reminder frequencies  
**FR-5.5** The system SHALL track reminder acknowledgment status

## 6 Decision Tracking

**FR-6.1** The system SHALL automatically detect decisions in conversations  
**FR-6.2** The system SHALL categorize decisions by type (Technical, Product, Process, etc.)  
**FR-6.3** The system SHALL extract decision context and stakeholders  
**FR-6.4** The system SHALL maintain a searchable decision log  
**FR-6.5** The system SHALL track decision implementation status
